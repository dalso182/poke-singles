# Vault Light theme (Angular Material 3)

Source of truth for the design spec: **`brand-guidelines.html`** at the repo root (open in a
browser — palette, typography, components, voice). The implementation in `src/styles/` mirrors
it. Brand summary: Tico Blue primary, Amber Glow tertiary, warm cream surfaces, Manrope + IBM
Plex Mono, density `-1`.

## Implementation files

| File | Role |
|---|---|
| `src/styles.scss` | Theme entry — runs `mat.theme()` + `mat.theme-overrides()` |
| `src/styles/_theme-colors.scss` | Material 3 tonal palettes — **generated**, don't hand-edit |
| `src/styles/_brand-tokens.scss` | `:root` CSS vars outside Material's color system |
| `src/styles/_material-overrides.scss` | Component overrides via M3 mixins |
| `src/styles/_brand-utilities.scss` | Brand utility classes |

- **`_theme-colors.scss`** is generated by
  `ng generate @angular/material:m3-theme`, seeded with `#1E3A8A` primary, `#D4941C` tertiary,
  `#B91C1C` error. Don't hand-edit — regenerate if the seeds change.
- **`_brand-tokens.scss`** holds `:root` vars for things outside Material's palette:
  `--brand-red`, `--accent-amber`, surfaces, semantic states, fonts.
- **`_material-overrides.scss`** uses M3 mixins (`mat.button-overrides`, `mat.card-overrides`):
  4px button radius, 8px card radius, uppercase 12px tracked button labels.
- **`_brand-utilities.scss`** provides `.brand-bar`, `.brand-eyebrow`, `.brand-mono`,
  `.product-card--on-sale`, `.product-card--sold-out`, `.price--sale`, `.price--original`.

`index.html` links Manrope + IBM Plex Mono and Material Icons.

## Hard rule: brand red

Brand red (`#CE1126`) is restricted to exactly three uses:

1. The brand-bar gradient (`.brand-bar`).
2. Sale prices / discount lines (`.price--sale`).
3. The AGOTADA / sold-out badge (`.product-card--sold-out::after`).

Brand red is **not** wired into Material's palette. Material's `error` slot uses **Danger
(`#B91C1C`)**, a different red, so form-field errors, snackbars, and chips never bleed brand
red. If brand red appears anywhere else, the wiring leaked — fix it before shipping.

(Note: amber is used for emphasis elsewhere; keep red reserved for the three cases above so
"out of stock" and the brand bar stay visually distinct from ordinary accents.)

## Verifying visual changes

The library page at **`/library`** (lazy, no shell wrapper) is a live reference: every Material
component the project uses, rendered with Vault Light. After any styling change, open `/library`
to confirm nothing drifted.

## Conventions

Material components are imported **per-component** via the `imports:` array (no shared barrel),
so component-level styling stays local; cross-cutting appearance goes through the theme files
above, not ad-hoc per-component CSS.

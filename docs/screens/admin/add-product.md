# Add product (admin)
> Part of the Poke-Singles docs set. Verified against source on 2026-07-16. Load together with /CLAUDE.md.

## Purpose
The TCGdex-driven product creation flow at `/admin/products/new`. Two entry modes: pick a card from the TCGdex typeahead (auto-fills name, rarity, number, image, variant, set, hidden metadata, and a suggested CRC price from TCGplayer market data), or `"Agregar sin TCGdex (accesorios u otro)"` for manual entry. Handles duplicate-SKU warnings, raffle products, card-type/sub-type assignment, consignment seller selection, auto-generated slugs, and batch entry (form resets for the next product after each create).

## Route & access
- Path: `/admin/products/new`, lazy `AddProduct` (`import('./admin/add-product/add-product')`), inside `AdminShell` behind `adminGuard` (see `src/app/app.routes.ts`).
- Registered **before** `products/:id/edit` in the route list, so `new` never matches as an `:id`.
- Query param: `?category=rifas` (used by the raffles screen's "Agregar rifa" deep link) preselects the Rifas category in `bootstrap()`, revealing the raffle section.
- Back link + post-create navigation targets: `/admin/products` (back header), `/admin/products/:id/edit` (dup-banner `goToProduct(id)` and the create snackbar's `Editar` action), `/admin/categories` (`goToCategories()` from the no-categories warning).

### Card typeahead (`CardTypeahead`)
Inputs `placeholder` (default `'Buscar cartasâ€¦'`), `setCode: string | null` (a TCGdex set id like `"sv05"`); output `cardSelected: Card`. `suggestions` = `toSignal` over `combineLatest([valueChanges, toObservable(setCode)])` with `filter(typeof value === 'string')` (Material writes the selected object back into the control), **`debounceTime(250)`**, `distinctUntilChanged()`, `startWith('')`; the switchMap returns `[]` for queries under **2 chars**, otherwise runs `Query.create().contains('name', q).paginate(1, 30)` via `tcgdex.client.card.list()` â€” once per set id when filtered (`.equal('set.id', id)`), in parallel over the base set **plus its gallery subsets**, results flattened base-first. Subset resolution: `subsetIdsFor(setCode)` memoizes `client.set.list()` and matches sets named exactly `base.name + ' Trainer Gallery' | ' Galarian Gallery' | ' Shiny Vault'` â€” by **name**, because TCGdex subset ids lie (`swsh12.5tg` is *Silver Tempest's* gallery, not Crown Zenith's; Hidden Fates' vault is `sma`). Errors swallow to `[]` (a failed set-list fetch clears the memo and degrades to base-set-only); `searching` signal drives a 20px suffix spinner. Options render a 28Ă—39 thumb (`${card.image}/low.webp`, skipped when the card has no image) + name + mono card id. `onSelect` fetches full detail via the low-level **`tcgdex.client.fetch('cards', resume.id)`** â€” not `card.get()` â€” because the model instance carries an `sdk` back-reference that makes `JSON.stringify` throw when the payload is cached to `card_details` (mirrors `scripts/seed-products.mjs`).

### Set typeahead (`SetTypeahead`)
Inputs `value` (Supabase set UUID), `placeholder` (default `'Buscar setâ€¦'`), `label` (default `'Set'`), `required`; output `valueChange: string | null`. Client-side filter (no server round trip) over `SetsService.list()` â€” matches diacritic-normalized `code`/`name`/`series`, capped at `MAX_SUGGESTIONS = 12`. Displays sets as `"{code} â€” {name}"` (`formatSet`). Free text is rejected: `onBlur()` snaps back to the current value's display or empty. A clear (`close`) suffix button (`aria-label="Quitar set"`) emits `null`. A constructor effect reloads the list if the bound `value` id isn't in the local list yet (e.g. a set just created by TCGdex hydration).

### Image picker dialog (`ImagePickerDialog`) â€” brief
Opened via `openImagePicker()`: `dialog.open(ImagePickerDialog, { width: '880px', maxWidth: '95vw', autoFocus: 'first-tabbable' })`; on close patches `image_url` with `result.url` and marks it dirty. Inside: breadcrumbs (root = `"RaĂ­z"`), folder/file grid, name filter (`debounceTime(150)`), upload (client-side checks: `"Solo se permiten imĂˇgenes."`, `MAX_UPLOAD_BYTES = 8 * 1024 * 1024` â†’ `"La imagen supera el lĂ­mite de 8 MB."`), inline new-folder creation. `ImageBrowserService` talks to the PHP endpoints deployed under the card-images root â€” `server/list-images.php` (GET `?path=`), `server/upload-image.php`, `server/create-folder.php`, all gated by `server/_supabase-auth.php` + `server/auth-config.php` and receiving the admin's Supabase session token via the `X-Supabase-Token` header (401/403 â†’ `"SesiĂłn expirada o sin permisos para imĂˇgenes."`). Endpoint configured at `environment.images.listUrl` (`'/card-images/list-images.php'`, root-relative so it rides the localhost dev proxy and stays same-origin in prod; empty string disables the picker â†’ `isEnabled()` false hides the suffix button). Returned absolute URLs are reduced to root-relative paths (`toRelativeUrl`) so picked images survive the domain cutover.

## Services & backend
- `TcgdexService.client` â€” `@tcgdex/sdk` `TCGdex('en')`; card search + detail fetch, set hydration.
- `TcgdexCardsService.upsert(card.id, card)` â†’ **`card_details`** table (`card_ref` PK, `data` jsonb, `fetched_at`). Called in `onCardSelected` so the `products.card_ref` FK is satisfied before insert; also read later by the public detail page. The table/column names are deliberately source-neutral (no "tcgdex" on the wire).
- `SetsService.findOrCreateFromTcgdex(card)` â€” `findByCode(card.set.id)` against `sets`; on miss hydrates the full set from TCGdex (`series`, `release_date`, `symbol_image_url` = `${symbol}.webp`, `printed_total` = `cardCount.official`) and inserts; on hit backfills a null `printed_total`. `SetsService.list()` is process-cached.
- `ProductsService`:
  - `listByCardRef(cardRef)` â€” `products` select (`id, slug, name, condition, variant, language, quantity, active, card_number, set_id`) by `card_ref`, newest first; admin RLS (`products_admin_all`) surfaces inactive rows.
  - `slugInUse(slug)` â€” head count on `products` by slug.
  - `create(input: ProductInsert)` â€” plain `products` insert returning `*`.
  - `setCardTypes(productId, ids)` â€” delete-then-insert on `product_card_types`.
- `RafflesService.upsert(productId, { draw_at, market_price })` â€” `raffles` upsert `onConflict: 'product_id'` (winner/status columns owned by `draw_raffle`).
- `AppSettingsService.get()` â€” fresh read of `app_settings`; only `exchange_rate_usd_crc` is used here.
- `CategoriesService.list({ activeOnly: true })`, `CardTypesService.list({ activeOnly: true })`, `SellersService.list({ activeOnly: true })` (name-ordered) â€” loaded in parallel in `bootstrap()`.
- `LocalStorageService.get/set` â€” persists the picker set filter.

- Form (`fb.nonNullable.group`, group validator `salePriceBelowPrice`): `name` (required), `pokemon_name`, `rarity`, `card_number`, `image_url`, `slug` (**disabled**, required, pattern `/^[a-z0-9-]+$/`), `set_id`, `category_id` (required), `seller_id`, `condition` (default `'NM'`), `language` (default `'EN'`, required), `variant`, `price` (default 0, required, min 0), `sale_price` (min 0.01), `quantity` (default 1, required, min 0), `featured`, `description`, `draw_at`, `market_price` (min 0), plus hidden TCGdex-derived controls never rendered as inputs: `card_ref`, `illustrator`, `regulation_mark`, `category`, `stage`, `type1`, `type2`, `legal_standard`, `legal_expanded` (patched by `onCardSelected`, serialised on submit; null in manual mode).
- Constructor: `bootstrap()`; a `form.valueChanges` subscription (via `takeUntilDestroyed`) calls `refreshSlug()` and mirrors `category_id` into `selectedCategoryId`; an `effect` writes `pickerSetId` to localStorage under `'admin:add-product:picker-set-id'` (cleared via the set-typeahead X).
- **Slug generation** (`computeSlug`, applied silently with `emitEvent: false`): card categories join `[name, card_number, setCode, variant, condition, langSuffix, sellerCode]`; others `[name, setCode, langSuffix, sellerCode]`. `langSuffix` only when language â‰  `'EN'`; `sellerCode` = seller's 2-char code lowercased (house inventory contributes nothing). Falsy parts dropped, joined `-`, lowercased, NFD diacritics stripped, non-alphanumerics collapsed to `-`, edges trimmed. `currentSlug` mirrors it so `exactDuplicate` re-evaluates live.
- **`onCardSelected(card)`**: sets `selectedCard`, resets `previewMissing`; fire-and-forget `listByCardRef(card.id)` â†’ `existingMatches` (errors â†’ `[]`, never blocks); patches name/pokemon_name/rarity/card_number, `image_url = tcgdexImageToHostedPath(card.image)` (e.g. `https://assets.tcgdex.net/en/swsh/swsh3/136` â†’ `/card-images/swsh/swsh3/136.webp`) â€” when `card.image` is absent (the SWSH gallery subsets), one `tcgdex.client.fetch('sets', card.set.id)` supplies the serie id and `hostedCardImagePath(serieId, setId, localId)` builds the same path (lookup failure â†’ blank field, admin uses the picker) â€” `variant = defaultVariantFor(card)` (prefers `normal`, else first true variant), and all hidden metadata; defaults `category_id` to the category with slug `singles` if none chosen; suggests `price = Math.ceil((usd * rate) / 100) * 100` from `tcgplayerMarketUsd(card)` Ă— `exchange_rate_usd_crc` (rounded **up** to the nearest â‚ˇ100; silently skipped when either is missing); upserts the raw payload into `card_details`; `findOrCreateFromTcgdex` resolves/creates the set, adds it to `setsById`, and patches `set_id`.
- **`onSubmit()`**: guard `form.invalid`; `slugInUse(raw.slug)` â†’ on conflict sets the `duplicate` error and snackbars `"Ese slug ya estĂˇ en uso. EdĂ­talo y reintenta."`; builds `ProductInsert` where non-card categories null out every card-derived column (`pokemon_name`, `rarity`, `variant`, `card_ref`, `illustrator`, `regulation_mark`, `category`, `stage`, `type1/2`, `legal_*`) **except** `card_number` and `condition`, which also persist when `isRaffle()` (raffle prizes are single cards); `price`/`quantity` coerced with `Number()`, `sale_price`/`market_price` via `toNullableNumber`. Then `setCardTypes` (`isCard` â†’ multi selection; subtype category â†’ the one `selectedSubtypeId`; else `[]`), `raffles.upsert` when raffle, snackbar `"Producto creado"` with action `"Editar"` â†’ the new product's edit page, and `resetForNext()`.
- **`resetForNext()`** clears everything but keeps the last `category_id` and `language` (batch entry) and exits manual mode. `enableManualMode()` fully resets (including category). `resetSelection()` (footer Cancelar / "Cambiar carta") returns to the picker without leaving the page.
- `previewSrc(value)` = `resolveHostedSrc(value, imageBrowser.origin)` â€” absolutizes `/card-images/...` paths against the picker endpoint's origin for the preview `<img>`; `previewMissing` flips on the img `error` event.

## Behaviors & edge cases
- Errors from bootstrap, `card_details` upsert, set find-or-create, and submit surface as snackbars with the thrown `message` (fallback `"Error desconocido"`), duration 5000 ms.
- The duplicate banner is **advisory only** â€” nothing blocks creating another SKU for the same `card_ref`; only an exact slug collision blocks submit.
- The suggested price never overwrites silently after the fact â€” it's patched once at pick time; the admin can type over it. No suggestion when the card lacks TCGplayer pricing or `exchange_rate_usd_crc` is unset/â‰¤ 0.
- `variantOptions` falls back to the full `VARIANT_OPTIONS` list if the card declares no variants or none are `true`.
- The set filter persists across page loads (localStorage) â€” an admin adding many cards from one set keeps the narrowing; a persisted UUID whose set row no longer resolves yields `pickerSetCode() === null` (unfiltered search).
- Card search requires â‰Ą 2 characters; TCGdex errors degrade to an empty suggestion list (no error UI).
- `?category=rifas` only works if a category with slug `rifas` exists.
- Products are created without an explicit `active` flag â€” the DB default applies (there is no Activo toggle here; deactivation lives in product-edit).

## Gotchas / invariants
- **The TCGdex cache table is `card_details`, not `tcgdex_cards`** (deliberately source-neutral so network traffic doesn't reveal the origin). Older docs/skills that say `tcgdex_cards` are stale.
- The `card_details` upsert must succeed before submit for card picks: `products.card_ref` is an FK to it. If the upsert failed (snackbar shown) and the admin submits anyway, the insert fails on the FK.
- The slug control is **disabled** â€” it is derived-only. Both duplicate-slug messages tell the admin to change *source* fields; the snackbar copy `"EdĂ­talo y reintenta."` is slightly misleading (the slug itself can't be typed into), the mat-error `"Slug ya en uso. Cambia uno de los campos de origen."` is the accurate one. In practice you change condition/variant/language/seller and the slug regenerates.
- `refreshSlug()` writes the slug with `emitEvent: false` to avoid a `valueChanges` loop â€” never remove that flag.
- Card detail fetch uses `client.fetch('cards', id)`, **not** `client.card.get(id)`: the model instance's `sdk` back-reference makes `JSON.stringify` throw a circular-structure error when caching. Keep it that way (mirrors `scripts/seed-products.mjs`).
- `firstTcgplayerVariant` picks the **first** non-metadata variant on `card.pricing.tcgplayer` â€” the suggested price and the TCGplayer link ignore which variant the admin actually selected (per-variant matching is a known deferred feature; same rule as the price-review report).
- Raffle exception: non-card categories clear card columns on insert, but `card_number` and `condition` are kept when the category is Rifas (they render on the /rifas tile). Raffle notes are stored in `products.description`; `draw_at`/`market_price` go to the `raffles` table via upsert.
- `ImageBrowserService.origin` is `''` when `listUrl` is root-relative (the default) â€” `new URL('/card-images/...')` throws, so `previewSrc` passes the relative path through unchanged. That's fine (same-origin + dev proxy) but means the preview only exercises the same host the admin UI runs on.
- Condition defaults to `'NM'`; for non-card, non-raffle categories the leftover `'NM'` is nulled at submit and dropped from the slug (the `isCard` recomputation inside `computeSlug` exists precisely so defaults like `NM` don't leak into non-card slugs).
- `existingMatches` is populated best-effort with `void`ed promise chains â€” a failed lookup silently shows no banner.
- Seller is **fixed at creation** (`ProductUpdate` excludes `seller_id`); the seller's lowercased 2-char code becomes the last slug part, disambiguating the same card across sellers.
- Amber, not brand red, for the duplicate banner (`--accent-amber*`) â€” the brand-red rule applies.
- **SWSH gallery subsets are separate TCGdex sets with no scans**: `swsh9.5tg`, `swsh11.5tg`, `swsh12.5tg`, `swsh12.5gg`, `swsh4.5sv`. The typeahead merges them into their base set's filter (by name, not id â€” ids are misleading); their image files come from the pokemontcg.io fallback in `scripts/fetch-card-images.mjs` (`PTCGIO_FALLBACK_SETS`), not TCGdex. Astral Radiance's TG cards are embedded in `swsh10` and need no special handling.
